  1. Monotonic Array

An array is monotonic if it is either monotone increasing or monotone decreasing.
An array A is monotone increasing if for all i <= j, A[i] <= A[j]. An array A is monotone decreasing if for all i <= j, A[i] >= A[j].
Return true if and only if the given array A is monotonic.
给出一个序列，判断是否是单调序列。
朴素想法：三个点a,b,c如果是单调序列(b-a),(c-b)的符号相同，否则相反。这里一定要保证a,b,c是互不相同的，不然会影响判断。因此用first,second,three记录a,b,c的下标，通过first = second;second = three;进行迭代。

  1. Bitwise ORs of Subarrays

We have an array A of non-negative integers.
For every (contiguous) subarray B = [A[i], A[i+1], …, A[j]] (with i <= j), we take the bitwise OR of all the elements in B, obtaining a result A[i] | A[i+1] | … | A[j].
Return the number of possible results. (Results that occur more than once are only counted once in the final answer.)
对数组A的所有连续子数组，求出子数组中所有值按位或的结果。连续子数组的意思是不能间隔，例如[1,2,4]有子数组[1],[2],[4],[1,2],[2,4],[1,2,4].本题我用hashset来保存之前计算过的“或值”，用迭代的方法算出所有集合。

  1. 由于hashset不能遍历且加入新值，因此代码中的set与tmpset搭配使用，计算新元素加入后得到的“或值”
  2. re负责记录整个过程中得到的结果
  3. 这个做法总是超时，原来在*的位置是写成了两行，先用整形变量计算出“或值”，再将该值加入集合，后来变成一句反而不超时了…不知道为什么
  4. 用clear也是为了避免总是申请新变量，减少开销。
