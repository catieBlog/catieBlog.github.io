904：只能收集两种水果

这个题目很长，总体意思就是连续子数组中有且仅有两个不同的数，问符合条件的最长连续子数组的长度。
我是用two pointer的思路做的，时间是300+ms，有其他的思路是用hashmap做的，大概100+ms，接下来分别说一下这两个思路。
首先，每次只排除一个候选人，因此统计候选人a，b最后一次出现的位置，取最小的作为下一次循环的起点。

但是这么做肯定比较耗费时间，再说一下two pointers的做法。hashmap用来存储两个候选人以及其分别的个数。右指针不断向后收集个数，当候选人的个数大于2时，左指针不断向前，知道其等于2，在这个过程中不断记录最大的个数（右指针-左指针）即可。

906 Super Palindromes

这个题是要求找到min，max范围内所有的回文数，并且要求其开根号后也是回文数。
为了避免多次开根号，可以先将左右范围开根号，只不过需要多次算平方；也可以直接找大回文数，开根号验证是否为回文数，只是还需再验证大回文数是否为平方数。
找规定范围内的回文数的函数是这么写的：对于一个core数可以变成两个回文数：奇数位和偶数位，如1->(1,11);12->(121,1221)，core从1开始不断增加，超过范围则结束计算，奇数位、偶数位分开算。判断是否为回文数的函数就更不用说了。

907 Sum of Subarray Minimums

Given an array of integers A, find the sum of min(B), where B ranges over every (contiguous) subarray of A.
Since the answer may be large, return the answer modulo 10^9 + 7.
Example 1:
Input: [3,1,2,4]
Output: 17
Explanation: Subarrays are [3], [1], [2], [4], [3,1], [1,2], [2,4], [3,1,2], [1,2,4], [3,1,2,4].
Minimums are 3, 1, 2, 4, 1, 1, 2, 1, 1, 1. Sum is 17.

Note:
1 <= A.length <= 30000
1 <= A[i] <= 30000

找出所有连续子数组的最小值的和，也就是1：先找出所有连续子数组，2：再找出每个子数组的最小值，3：求和。
这么做肯定会超时，换一个思路，对于A[i]它能成为几个连续子数组的最小值呢？如果A[i]左边有l个连续的数比A[i]小，右边有r个数比A[i]大，那么就有l*r个连续子数组的最小值是A[i]。查找有左右连续的较大数是有原题的，leetcode42。

discuss里还有用单调栈做的，思路一样，用左右边界数组的方法更快一点。
